### Phase 8-A: DFS クローラー安定化 (back() バグ修正)

#### 問題
3分統合テストで '設定'(Settings ROOT) が depth=1 の子として検出された。
`_crawl_impl()` が戻り値を返さず (`None`)、非遷移タップ後でも `back()` が実行されていた。

#### 修正内容
- `_crawl_impl()` の戻り型を `None → bool` に変更
  - `True` → 画面遷移あり → 親が `back()` を実行
  - `False` → 同一画面（`fingerprint == parent_fp`）→ `back()` スキップ、`[NO_NAV]` ログ
- `crawl()` ループで `_did_navigate` を受け取り、条件付き `back()` 呼び出し

### Phase 8-C: Web 管理画面 — 実探索データ接続

#### 問題

`crawler/evidence/` に 6 セッション・20 画面の実データが蓄積されているが活用できていなかった。

#### 解決策
PHP PDO SQLite が利用可能 (`bool(true)`) → SQLite フォールバックを実装
